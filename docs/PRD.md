# PRD 

## 개요

* Express 하나로 정적 파일(프론트)과 REST API(백엔드)를 동시에 제공
* 아래 기능을 제공하는 Milvus 관리 사이트를 작성해줘.
* 페이지 이름은 자연스러운 영어로 변경하고, API 등 주소도 영어로 작성해줘.
* 네비게이션 구조도 자연스러운 형태로 수정해서 메뉴체계를 구축해줘.


## 동적설계

```navigation
Home --> 컬렉션세부정보
컬렉션세부정보 --> 백업
컬렉션세부정보 --> 삭제
컬렉션세부정보 --> 비우기
Home --> 컬렉션생성
Home --> 컬렉션복구
Home --> 컬렉션조회
컬렉션조회 --> 레코드수정
```

* Home: 컬렉션 목록을 표시한다.
* 컬렉션세부정보: 선택된 컬렉션의 정보를 표시한다. (컬럼 목록, 타입, 인덱스 타입 등 세부 정보)
* 백업: 해당 컬렉션의 데이터를 jsonl로 저장한다. 이때 스키마 구조는 json으로 저장한다. "날짜-컬렉션이름.jsonl"
* 삭제: 해당 컬렉션을 삭제한다.
* 비우기: 해당 컬렉션의 레코드만 삭제한다.
* 컬렉션생성: 스키마 정보를 입력하면 컬렉션을 생성한다. "날짜-컬렉션이름.json"
* 컬렉션복구: 저장된 레코드를 해당 컬렉션에 추가한다. "날짜-컬렉션이름.jsonl"
* 컬렉션조회: 도표형식으로 레코드를 출력한다. 페이징이 필요하다.
* 레코드수정: 선택한 레코드의 내용을 수정한다.

## 코딩 지침

* 가독성 우선
  - 누구나 빠르게 읽고 이해할 수 있게 단순·명확하게 작성한다.
* 단일 책임
  - 하나의 파일은 하나의 책임만 가진다.
  - 기능·책임 단위로 파일을 분리한다.
* 탑다운 구조
  - 메인 파일은 전체 흐름만 드러낸다.
  - 상세 로직과 구현은 별도 파일(모듈/클래스/함수)로 분리한다.
